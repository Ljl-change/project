# Aurora Player

Aurora Player 是一个基于 `PyQt6` 的桌面音乐播放器项目，当前聚焦在本地音乐播放、歌单管理、歌词展示、封面提取和测试直播流播放这几个核心能力上。


## 功能特性

- 本地音乐播放：支持 `mp3`、`ogg`、`wav`、`flac`、`m4a`、`aac`
- 歌单管理：新建、重命名、删除、保存到本地
- 曲目管理：添加歌曲、导入整个文件夹、删除歌曲
- 播放模式：列表循环、单曲循环、随机播放
- 搜索过滤：按歌曲名、歌手、专辑、文件名过滤当前歌单
- 拖拽排序：在完整歌单视图中拖拽调整曲目顺序
- 元数据读取：自动读取标题、歌手、专辑、时长
- 封面提取：提取并缓存专辑封面
- 本地歌词：读取同名 `.lrc` 文件并显示
- 播放状态持久化：保存音量、窗口大小、上次歌单、上次曲目、上次进度
- 直播页：支持加载示例 HLS 流并播放测试直播源
- 快捷键支持：`Ctrl+F`、`Ctrl+1/2/3`、`Space`、`Ctrl+Left/Right`

## 技术栈

- Python 3.9+
- PyQt6
- Pillow
- mutagen

说明：
- `mutagen` 用于读取音频元数据和封面
- 当前代码已经兼容“未安装 mutagen 也能启动”，但建议安装完整依赖以获得完整功能

## 项目结构

```text
.
├─ main.py
├─ app/
│  ├─ controllers/     # 页面逻辑与交互控制
│  ├─ models/          # Playlist / Track / StreamItem / AppSettings
│  ├─ repositories/    # 本地 JSON 数据读写
│  ├─ services/        # 播放、元数据、歌词、设置、直播等服务
│  ├─ ui/              # 主窗口、页面和组件
│  ├─ utils/           # 常量、路径、时间等工具
│  └─ workers/         # 预留给后台任务
├─ assets/
│  └─ demo_streams.json
└─ data/
   ├─ playlists/
   ├─ cache/
   └─ settings.json
```

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 运行项目

```bash
python main.py
```

## 数据说明

项目运行后会在本地生成和维护这些数据：

- `data/settings.json`
  保存音量、播放模式、窗口大小、上次播放状态
- `data/playlists/*.json`
  保存歌单与曲目信息
- `data/cache/covers/`
  缓存提取后的专辑封面

当前已经加入基础容错逻辑：
- 当 `settings.json` 丢失时，程序会自动补回默认配置
- 当配置内容损坏或字段缺失时，会按默认值自动修复

## 当前完成度

当前版本更适合作为“可继续进化的桌面播放器基础版”，已经具备完整的本地播放主链路：

- 能导入和组织本地音乐
- 能保存歌单和播放状态
- 能显示基础元数据、封面和歌词
- 能播放示例直播流
- UI 已经完成统一的桌面布局和主题风格

## 后续优化方向

### 1. 完善异常处理

目前本地 JSON、音频文件、直播链接读取的异常处理还可以继续加强，例如：

- 歌单文件损坏时自动跳过并提示
- 音频文件不存在时给出更清晰的反馈
- 直播地址无法播放时显示错误原因

### 2. 引入后台扫描任务

当前文件夹导入是同步扫描。后续可以把大目录扫描放到 `workers/scan_worker.py`，避免导入大量文件时界面卡顿。

### 3. 补齐播放体验

可以继续增加：

- 拖动进度条时的实时预览
- 播放历史与最近播放
- 收藏歌曲
- 单曲信息面板
- 退出时记住更多 UI 状态

### 4. 提升歌词能力

当前歌词展示还是本地 `.lrc` 文本读取，可以继续增强：

- 解析时间轴
- 实时滚动高亮
- 多编码歌词兼容
- 无歌词时的友好占位提示

### 5. 提升直播页完整度

直播页目前更接近“测试流播放器”，后续可以加上：

- 自定义流列表持久化
- 地址可用性检测
- 缓冲状态展示
- 最近播放直播记录

### 6. 增加测试

建议先从最稳定的纯逻辑层开始补测试：

- `playlist_service.py`
- `settings_repository.py`
- `library_controller.py`
- `lyric_service.py`

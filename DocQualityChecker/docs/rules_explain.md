# DocQualityChecker 检测规则说明

本文档用于解释 DocQualityChecker 当前版本采用的主要检测规则、规则原理、适用场景与局限性，方便用户理解检测结果，也便于后续持续验证工具可靠性。

> DocQualityChecker 当前采用启发式规则检测，适合用于提交前自查和格式初筛。检测结果具有参考价值，但不能替代人工审阅、正式查重、排版软件或专业审稿流程。

## 1. 文档结构完整性检测

检测目标：
判断文档是否包含摘要、关键词、引言/绪论、结论/总结、参考文献等常见章节。

检测原理：
通过标题文本和全文关键词进行启发式匹配，判断目标章节是否出现。

严重程度判断：
- 缺失目标章节时，通常记为 `warning`

可信度说明：
- 中等
- 对规范命名的论文、课程报告、比赛文档较可靠

可能误报情况：
- 文档类型不是论文或报告
- 用户使用了非常规标题，例如“结束语”“工作展望”

示例：
- 未检测到“参考文献”部分
- 未检测到“关键词”部分

## 2. 段落长度检测

检测目标：
识别过长段落、过短段落和连续空段。

检测原理：
- 对每个段落去除空白字符后统计长度
- 超过阈值视为过长段落
- 非标题且长度过短时提示内容可能不足
- 连续空段超过 2 个时提示空行过多

严重程度判断：
- 过长段落：`warning`
- 过短段落：`info`
- 连续空段：`info`

可信度说明：
- 过长段落：高
- 过短段落：中等
- 连续空段：高

可能误报情况：
- 公式、编号说明或短注释可能天然较短
- 某些技术文档允许较长段落连续说明

示例：
- 单段长度为 520，超过阈值 300
- 连续空段超过 2 个

## 3. 未完成标记检测

检测目标：
识别提交前仍残留的 TODO、待补充、XXX、这里写等占位文本。

检测原理：
对常见未完成标记进行关键词匹配，只要命中即提示。

严重程度判断：
- 通常记为 `warning`

可信度说明：
- 高
- 这类问题大多由明确文本直接触发

可能误报情况：
- 仅在个别特殊说明文档中，关键词可能被当作示例文本

示例：
- 检测到未完成标记“TODO”
- 检测到未完成标记“待补充”

## 4. 标题层级检测

检测目标：
识别标题过长、标题只有编号、标题跳级、同级编号不连续等问题。

检测原理：
- 结合标题样式名和编号形式识别标题
- 对数字标题编号进行层级和连续性启发式检查
- 对标题编号后的实际文本进行判断，识别空标题

严重程度判断：
- 标题过长：`info`
- 只有编号无标题内容：`warning`
- 明显跳级：`warning`
- 同级编号不连续：`warning`

可信度说明：
- 空标题：高
- 跳级和不连续：中等
- 标题过长：中等

可能误报情况：
- 文档采用了特殊编号体系
- 某些标题可能故意省略中间层级

示例：
- 从 `1` 直接跳到 `1.1.1`
- `1.1` 后直接出现 `1.3`
- 标题只有“1.2”而没有实际内容

## 5. 图表编号检测

检测目标：
识别图表编号重复、图表编号不连续，以及引用图但附近没有明确图号的情况。

检测原理：
- 通过文本模式匹配提取 `图1`、`图1-1`、`图 1.1`、`表1` 等形式
- 对纯数字编号进行简单连续性判断
- 对“如下图所示”“见下图”等表述做邻近段落检查

严重程度判断：
- 编号重复：`warning`
- 编号不连续：`warning`
- 引用图但附近未检测到图号：`info`

可信度说明：
- 编号重复：高
- 编号不连续：中等
- 无图号引用：低

可能误报情况：
- 图表采用章节式编号时，连续性规则不会完全覆盖
- 图题与正文距离较远时，邻近检查可能误判

示例：
- 图1 重复出现两次
- 图1、图2、图4
- 正文写“如图所示”，但附近没有 `图X`

## 6. 基础格式检测

检测目标：
识别多空格、重复标点、中英文标点混用、括号不匹配等基础格式问题。

检测原理：
- 用正则表达式检测连续空格和连续标点
- 统计中文段落中的英文逗号或句号数量
- 分别统计中英文括号左右数量是否匹配

严重程度判断：
- 多空格：`info`
- 重复标点：`info`
- 中英文标点混用：`info`
- 括号不匹配：`warning`

可信度说明：
- 括号不匹配：高
- 多空格与重复标点：高
- 标点混用：中等

可能误报情况：
- 示例代码、英文材料或引用内容中可能天然存在英文标点

示例：
- 出现 `，，`
- 中文段落中出现大量 `,` 和 `.`
- `（` 数量与 `）` 不一致

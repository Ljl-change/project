# DocQualityChecker 使用说明

本文档用于帮助你快速上手 DocQualityChecker，包括命令行模式、Streamlit 可视化界面、常见问题和结果查看方式。

## 1. 项目简介

DocQualityChecker 是一个用于 `.docx` 文档自检的小工具，可以在提交论文、课程报告、比赛材料、项目文档之前，快速发现以下常见问题：

- 文档结构缺失
- 段落过长或过短
- 标题层级或编号异常
- 图表编号重复或不连续
- TODO、待补充等未完成标记残留
- 多空格、重复标点、括号不匹配等基础格式问题

## 2. 环境要求

- Python 3.9 及以上
- 支持 Windows / Linux / macOS

安装依赖：

```bash
pip install -r requirements.txt
```

## 3. 两种使用方式

### 3.1 命令行模式

适合开发者、本地批处理、脚本调用。

运行示例：

```bash
python main.py --file examples/sample.docx
```

指定报告输出路径：

```bash
python main.py --file examples/sample.docx --output outputs/report.md
```

修改最大段落长度阈值：

```bash
python main.py --file examples/sample.docx --max-paragraph-length 300
```

查看帮助：

```bash
python main.py --help
```

### 3.2 可视化界面模式

适合普通用户直接上传文档、查看图表和下载报告。

启动方式：

```bash
streamlit run app.py
```

启动后终端通常会显示：

```text
Local URL: http://localhost:8501
```

然后在浏览器打开：

```text
http://localhost:8501
```

注意：

- 终端里显示的是启动日志，不是前端界面本身
- 真正的 UI 界面在浏览器里打开
- 第一次启动 Streamlit 时如果提示输入邮箱，直接回车跳过即可

## 4. 前端界面使用步骤

1. 启动前端：`streamlit run app.py`
2. 浏览器打开 `http://localhost:8501`
3. 上传一个 `.docx` 文件
4. 在左侧边栏设置参数
5. 点击 `开始检测`
6. 查看评分、结构检查、问题列表和问题统计
7. 下载 Markdown 检测报告

## 5. 参数说明

### 最大段落长度阈值

含义：

- 一个段落在去除空白字符后，如果字符数超过这个值，就会被提示为“段落过长”

示例：

- 阈值设为 `300`
- 某段长度为 `280`，不会提示过长
- 某段长度为 `350`，会提示过长

建议值：

- 论文、课程报告：`300`
- 希望检查更严格：`200-250`
- 技术说明、连续叙述较多：`350-500`

### 是否显示原文片段

- 开启后，问题列表会展示部分原文片段，便于快速定位
- 关闭后，页面更简洁

### 是否生成 Markdown 报告

- 开启后，会自动生成 Markdown 报告并提供下载
- 关闭后，只在页面中展示检测结果

## 6. 检测结果怎么看

### 综合评分

评分范围是 `0-100`，初始分为 `100`，根据问题严重程度扣分：

- `error`：每个扣 8 分
- `warning`：每个扣 3 分
- `info`：每个扣 1 分

等级划分：

- `90-100`：优秀
- `75-89`：良好
- `60-74`：一般
- `60 以下`：建议修改

说明：

- 该评分仅用于提交前自查
- 不能替代正式审稿、查重或人工校对

### 结构完整性检查

会检查以下常见章节是否存在：

- 摘要
- 关键词
- 引言 / 绪论
- 结论 / 总结
- 参考文献

### 问题列表

问题列表支持按以下维度筛选：

- 严重程度：`error / warning / info`
- 问题类型：`structure / paragraph / heading / figure_table / format / unfinished`

## 7. 输出文件位置

### 命令行模式

- 默认报告路径：`outputs/check_report.md`

### Streamlit 前端模式

- 上传文件保存目录：`outputs/uploads/`
- 前端生成报告目录：`outputs/streamlit_reports/`

文件名会自动带时间戳，避免覆盖旧文件。

## 8. 示例文档

仓库已经自带一份示例文档：

```text
examples/sample.docx
```

你可以直接运行：

```bash
python main.py --file examples/sample.docx
```

或在前端界面中上传该文件查看效果。

这份示例文档故意包含了若干问题，方便演示：

- 缺失“参考文献”章节

- 图表编号重复或不连续
- TODO / 待补充类占位内容
- 基础格式问题

## 9. 让同一局域网的别人访问

如果你希望同一网络下的其他电脑访问你的前端，可以这样启动：

```bash
streamlit run app.py --server.address 0.0.0.0 --server.port 8501
```

然后把你的局域网地址发给对方，例如：

```text
http://10.190.214.212:8501
```

注意需要满足以下条件：

- 对方和你在同一个局域网
- 你的电脑没有关机
- Streamlit 程序仍在运行
- 防火墙没有拦截 8501 端口

## 10. 常见问题

### 1. 启动后为什么终端里有输出，不是图形界面？

因为 `streamlit run app.py` 是在终端里启动服务，真正的图形界面会在浏览器中打开。

### 2. 第一次启动为什么提示输入 Email？

这是 Streamlit 的默认提示，不是项目本身在收集邮箱。直接回车跳过即可。

### 3. 为什么上传后检测失败？

请优先检查：

- 文件是否为 `.docx`
- 文件是否损坏
- 文件是否确实包含正文内容
- 是否已经正确安装依赖

### 4. 为什么别人打不开我的局域网地址？

通常原因有：

- 不在同一个网络
- 防火墙阻止访问
- 没有使用 `--server.address 0.0.0.0`
- 程序已经关闭

## 11. 免责声明

- 正式论文审稿
- 学术查重
